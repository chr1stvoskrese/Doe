# Doe для iOS

Нативное SwiftUI-приложение (iOS 17+), работающее с тем же файлом-хранилищем
`*.db.doe`, что и десктоп-версия Doe. Открываешь папку в iCloud Drive — и
работаешь с доской и на Mac, и на iPhone, как в Obsidian.

## Что внутри (первый минимальный срез)

- Колонки и карточки канбана, полная адаптивность под все экраны (вкл. iPhone 16).
- Markdown в описании карточки: заголовки, списки, цитаты, код, **ссылки**,
  изображения из подпапки `doe/`. Редактор + живой просмотр.
- Подзадачи по графу **многие-ко-многим (All-to-All)**: у карточки может быть
  несколько родителей и детей, переходы между ними, защита от циклов.
- Модалка карточки повторяет десктоп, **кроме** блоков «запоминание» и
  «инструменты» (как и просили).
- Вкладки **без сворачивания**, переключение и перетаскивание для переупорядочивания.
- Анимации перетаскивания: карточек внутри колонки, между колонками и между
  вкладками (drop карточки на таб), а также перетаскивание самих колонок.
- Экран выбора хранилища: открыть папку, создать новую, список недавних.
- Локальные напоминания (системные уведомления iOS).

Намеренно **не** реализованы (по договорённости): поиск, дедлайны, таймеры-UI,
приоритеты, AI, автоматизации, статистика, запоминание. База данных полностью
совместима с этими функциями — они продолжат работать на десктопе.

## Совместимость данных (важно)

Схема БД снята дословно с десктопа и проверена: новый vault, созданный на iPhone,
содержит все таблицы, индексы, FTS5-поиск и триггеры, а `alembic_version`
проставлен в head-ревизию `m1n2e3m4o5r6`. Реальные миграции десктопа,
запущенные против такой базы, видят её как уже обновлённую (no-op, без ошибок).

Формат значений совпадает с SQLAlchemy/SQLite десктопа:
- даты: `YYYY-MM-DD HH:MM:SS.ffffff` (наивный UTC),
- `mode` колонок: `DEFAULT` / `TRACK_TIME` / `COMPLETION`,
- булевы: `0/1`, JSON-поля: текст (`[]`).

Если открыть на iPhone папку с базой **старой** ревизии, приложение не станет с
ней работать и попросит сначала один раз открыть её в свежей десктоп-версии,
чтобы Alembic обновил схему. Так ничего не ломается.

## Как открыть и собрать

1. Открой `iOS/Doe.xcodeproj` в **Xcode 16+** (проект использует синхронизацию
   с папкой `Doe/`, поэтому новые файлы подхватываются автоматически).
2. Выбери таргет **Doe**, в *Signing & Capabilities* укажи свою команду (Team).
   Личного Apple ID достаточно для запуска на своём устройстве.
3. Для SQLite используется системная библиотека (`-lsqlite3`), внешних
   Swift-пакетов нет — резолвить зависимости не нужно.
4. Запусти на симуляторе или устройстве (iOS 17+).

## Как пользоваться синхронизацией (как в Obsidian)

1. На Mac положи или создай хранилище Doe в папке iCloud Drive.
2. На iPhone: экран выбора → **Открыть папку** → перейди в iCloud Drive и выбери
   ту же папку. Либо **Создать хранилище** прямо в iCloud Drive, затем открой эту
   же папку на Mac.
3. Изменения подхватываются автоматически: приложение отслеживает файл и
   перечитывает доску при внешних правках и при возврате на передний план.

Рекомендация: не редактируй одно хранилище одновременно на двух устройствах —
iCloud синхронизирует файл целиком, и одновременные правки могут конфликтовать
(то же ограничение, что у однофайловой базы на десктопе).

## Структура кода

```
Doe/
  DoeApp.swift            — точка входа, маршрутизация picker ↔ доска
  Models/Models.swift     — Workspace, Column, Card, ColumnMode
  Storage/
    SQLite.swift          — обёртка над libsqlite3
    Schema.swift          — точная схема head-ревизии + формат дат
    VaultStore.swift      — репозиторий (CRUD, граф подзадач, перемещение)
    VaultManager.swift    — iCloud-папка, bookmarks, отслеживание изменений
  ViewModels/BoardViewModel.swift — состояние доски, drag-операции
  Views/                  — доска, колонки, карточки, модалка, выбор хранилища
  Support/                — тема, уведомления, типы перетаскивания
```

## Примечание

В каталогах могут встретиться скрытые файлы `.fuse_hidden*` — это артефакт
среды, где генерировался проект. Они не имеют расширения `.swift`, Xcode их
игнорирует; их можно спокойно удалить.
